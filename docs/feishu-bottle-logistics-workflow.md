# 飞书多维表格：客户寄回/补发瓶子物流与仓库确认流程搭建说明

## 1. 目标与适用场景

本多维表格用于统一管理客户寄回瓶子、仓库收货确认、办公人员补发瓶子的全流程，解决以下问题：

- 客户提供快递单号后，办公人员可及时录入并跟进物流。
- 仓库人员可在签收后确认实际收到或寄出的瓶子数量。
- 系统可通过公式自动判断数量是否一致、是否待仓库确认、是否待办公人员发货、是否完成。
- 办公人员与仓库人员各自只看自己需要处理的视图，减少漏处理和重复沟通。

## 2. 数据表建议

建议创建一个飞书多维表格，表名为：`瓶子寄回与补发跟进表`。

所有记录统一放在一张主表中，通过字段和视图区分不同处理阶段，便于筛选、统计和追溯。

## 3. 字段设计

| 字段名称 | 字段类型 | 是否必填 | 建议选项 / 规则 | 用途说明 |
| --- | --- | --- | --- | --- |
| 客户姓名/店铺名 | 文本 | 是 | 手动填写 | 记录客户姓名、门店名或店铺名。 |
| 客户联系方式 | 电话 / 文本 | 是 | 手机号、微信号或其他联系方式 | 便于异常时联系客户确认。 |
| 订单号 | 文本 | 建议必填 | 可填写电商订单号、内部订单号 | 便于和原始订单关联。 |
| 快递公司 | 单选 | 是 | 顺丰、京东、圆通、中通、韵达、申通、极兔、德邦、其他 | 记录客户寄回或补发使用的物流公司。 |
| 快递单号 | 文本 | 是 | 手动填写 | 客户提供快递单号后录入。 |
| 客户反馈需要邮寄的瓶子数量 | 数字 | 是 | 仅允许整数，最小值 0 | 客户反馈应寄回或需补发的瓶子数量。 |
| 实际仓库收到/寄出的瓶子数量 | 数字 | 仓库确认时必填 | 仅允许整数，最小值 0 | 仓库根据实物清点后填写。 |
| 物流状态 | 单选 | 是 | 待查询、运输中、已签收、异常 | 由办公人员手动更新，或后续接入物流查询自动更新。 |
| 签收时间 | 日期时间 | 条件必填 | 物流状态为已签收时填写 | 记录物流签收时间。 |
| 仓库确认状态 | 单选 | 是 | 未确认、数量正确、数量不符、异常 | 仓库清点后选择；也可结合公式字段辅助判断。 |
| 办公人员发货状态 | 单选 | 是 | 未发货、已发货 | 办公人员根据是否完成补发更新。 |
| 办公人员勾选“已发货” | 复选框 | 否 | 勾选代表已完成发货动作 | 便于办公人员用勾选动作快速完成发货。 |
| 备注 | 多行文本 | 否 | 手动填写 | 记录异常原因、沟通记录、补发说明等。 |
| 操作人 | 人员 | 建议必填 | 选择最后处理人员 | 记录最近一次处理人或责任人。 |
| 更新时间 | 最后修改时间 | 自动 | 自动记录 | 用于排序和追踪最近更新。 |
| 数量核对结果 | 公式 | 自动 | 见公式建议 | 自动判断客户反馈数量与仓库实际数量是否一致。 |
| 流程阶段 | 公式 / 单选 | 自动优先 | 见公式建议 | 自动汇总当前记录所处阶段，供视图筛选使用。 |

## 4. 推荐公式字段

### 4.1 数量核对结果

字段名：`数量核对结果`  
字段类型：`公式`

推荐逻辑：

```text
IF(
  ISBLANK({实际仓库收到/寄出的瓶子数量}),
  "待仓库填写数量",
  IF(
    {实际仓库收到/寄出的瓶子数量} = {客户反馈需要邮寄的瓶子数量},
    "一致",
    "不一致"
  )
)
```

使用说明：

- 仓库尚未填写实际数量时，显示 `待仓库填写数量`。
- 两个数量一致时，显示 `一致`。
- 两个数量不一致时，显示 `不一致`，办公人员应查看备注或联系仓库/客户复核。

### 4.2 流程阶段

字段名：`流程阶段`  
字段类型：`公式`

推荐逻辑：

```text
IF(
  {物流状态} = "异常",
  "数量不符/异常",
  IF(
    {仓库确认状态} = "异常",
    "数量不符/异常",
    IF(
      {仓库确认状态} = "数量不符",
      "数量不符/异常",
      IF(
        AND({物流状态} = "已签收", {仓库确认状态} = "未确认"),
        "已签收待仓库确认",
        IF(
          AND({仓库确认状态} = "数量正确", NOT({办公人员勾选“已发货”})),
          "待办公人员发货",
          IF(
            AND({仓库确认状态} = "数量正确", {办公人员勾选“已发货”}),
            "已完成记录",
            IF(
              OR({物流状态} = "待查询", {物流状态} = "运输中"),
              "待查询物流",
              "待处理"
            )
          )
        )
      )
    )
  )
)
```

使用说明：

- 物流异常、仓库异常或数量不符时，优先进入 `数量不符/异常`。
- 物流已签收但仓库未确认时，自动进入 `已签收待仓库确认`。
- 仓库确认数量正确但办公人员未勾选已发货时，进入 `待办公人员发货`。
- 仓库确认数量正确且办公人员已勾选已发货时，进入 `已完成记录`。
- 物流待查询或运输中时，进入 `待查询物流`。

## 6. 状态流转逻辑

```text
客户提供快递单号
  ↓
办公人员录入客户信息、订单号、快递公司、快递单号、客户反馈数量
  ↓
物流状态 = 待查询 / 运输中
  ↓
视图：待查询物流
  ↓
办公人员查询物流并更新物流状态
  ↓
物流状态 = 已签收，填写签收时间
  ↓
仓库确认状态仍为 未确认
  ↓
系统自动进入：已签收待仓库确认
  ↓
仓库人员清点实物并填写实际仓库收到/寄出的瓶子数量
  ↓
仓库人员更新仓库确认状态
  ├─ 数量不一致或有异常 → 仓库确认状态 = 数量不符 / 异常 → 进入：数量不符/异常
  └─ 数量一致 → 仓库确认状态 = 数量正确 → 进入：待办公人员发货
        ↓
      办公人员完成发货
        ↓
      勾选 办公人员勾选“已发货”，并将办公人员发货状态改为 已发货
        ↓
      系统自动进入：已完成记录
```

## 7. 自动化建议

如希望减少手动维护，可以在飞书多维表格中配置以下自动化：

### 7.1 签收后提醒仓库确认

触发条件：

- 当 `物流状态` 变为 `已签收`
- 且 `仓库确认状态` 等于 `未确认`

执行动作：

- 发送飞书消息给仓库负责人。
- 消息内容包含客户姓名/店铺名、订单号、快递公司、快递单号、客户反馈数量和记录链接。

### 7.2 仓库确认数量不符时提醒办公人员

触发条件：

- 当 `仓库确认状态` 变为 `数量不符` 或 `异常`
- 或 `数量核对结果` 变为 `不一致`

执行动作：

- 发送飞书消息给办公人员或客服主管。
- 提醒查看备注并联系客户或仓库复核。

### 7.3 数量正确后提醒办公人员发货

触发条件：

- 当 `仓库确认状态` 变为 `数量正确`
- 且 `办公人员勾选“已发货”` 未勾选

执行动作：

- 发送飞书消息给办公人员。
- 提醒处理补发或后续发货动作。

### 7.4 勾选已发货后同步发货状态

触发条件：

- 当 `办公人员勾选“已发货”` 被勾选

执行动作：

- 自动将 `办公人员发货状态` 更新为 `已发货`。

## 8. 办公人员操作流程

### 8.1 客户提供快递单号后

1. 打开 `瓶子寄回与补发跟进表`。
2. 新增一条记录。
3. 填写以下字段：
   - 客户姓名/店铺名
   - 客户联系方式
   - 订单号
   - 快递公司
   - 快递单号
   - 客户反馈需要邮寄的瓶子数量
   - 物流状态：选择 `待查询` 或 `运输中`
   - 仓库确认状态：选择 `未确认`
   - 办公人员发货状态：选择 `未发货`
   - 操作人
   - 备注，如有特殊说明
4. 后续在 `待查询物流` 视图中持续跟进物流。

### 8.2 查询到物流已签收后

1. 将 `物流状态` 改为 `已签收`。
2. 填写 `签收时间`。
3. 确认 `仓库确认状态` 仍为 `未确认`。
4. 记录会自动进入 `已签收待仓库确认` 视图，等待仓库处理。

### 8.3 仓库确认数量正确后

1. 打开 `待办公人员发货` 视图。
2. 完成实际补发或发货动作。
3. 勾选 `办公人员勾选“已发货”`。
4. 将 `办公人员发货状态` 改为 `已发货`；如已配置自动化，则可由系统自动修改。
5. 在 `备注` 中填写补发说明、补发单号或其他必要信息。
6. 记录会自动进入 `已完成记录` 视图。

### 8.4 遇到数量不符或异常时

1. 打开 `数量不符/异常` 视图。
2. 查看 `数量核对结果`、`仓库确认状态` 和 `备注`。
3. 联系仓库复核实物数量，必要时联系客户确认寄出数量。
4. 在 `备注` 中记录处理过程和结论。
5. 复核后如确认数量正确，将 `仓库确认状态` 改为 `数量正确`。
6. 如仍异常，保持 `数量不符` 或 `异常`，并持续跟进。

## 9. 仓库人员操作流程

### 9.1 每日处理待确认包裹

1. 打开 `已签收待仓库确认` 视图。
2. 按 `签收时间` 从早到晚处理。
3. 根据快递单号找到对应包裹。
4. 清点实际收到或需要寄出的瓶子数量。
5. 填写 `实际仓库收到/寄出的瓶子数量`。
6. 对比 `数量核对结果`。

### 9.2 数量一致时

1. 将 `仓库确认状态` 改为 `数量正确`。
2. 在 `备注` 中填写“仓库已确认，数量正确”等说明。
3. 更新 `操作人` 为当前仓库人员。
4. 记录会自动进入 `待办公人员发货` 视图。

### 9.3 数量不一致时

1. 将 `仓库确认状态` 改为 `数量不符`。
2. 在 `备注` 中写清楚实际情况，例如：
   - “客户反馈 20 个，仓库实收 18 个。”
   - “外箱破损，疑似运输途中丢失。”
   - “客户混寄其他物品，需要办公人员确认。”
3. 更新 `操作人` 为当前仓库人员。
4. 记录会自动进入 `数量不符/异常` 视图。

### 9.4 包裹或物流异常时

1. 如发现包裹破损、空包、单号不匹配、无法找到包裹等情况，将 `仓库确认状态` 改为 `异常`。
2. 在 `备注` 中写清楚异常描述。
3. 必要时上传照片到飞书附件字段；如需要该能力，可额外增加字段 `异常照片`，类型为附件。
4. 通知办公人员或等待自动化提醒。

## 10. 权限建议

建议按角色配置权限：

| 角色 | 建议权限 |
| --- | --- |
| 办公人员 | 可新增记录，可编辑客户信息、物流状态、签收时间、发货状态、备注。 |
| 仓库人员 | 可查看已签收待仓库确认记录，可编辑实际数量、仓库确认状态、备注、操作人。 |
| 主管 / 管理员 | 可查看全部记录，可编辑全部字段，可维护视图、字段和自动化。 |

如果飞书版本支持字段级权限，建议限制仓库人员不要误改客户联系方式、订单号和快递单号；限制办公人员不要误改仓库实际数量。

## 11. 搭建步骤清单

1. 新建飞书多维表格，命名为 `瓶子寄回与补发跟进表`。
2. 按“字段设计”章节逐一创建字段。
3. 为以下字段配置单选选项：
   - 快递公司
   - 物流状态
   - 仓库确认状态
   - 办公人员发货状态
4. 创建公式字段：
   - 数量核对结果
   - 流程阶段
5. 创建以下视图并配置筛选条件：
   - 待查询物流
   - 已签收待仓库确认
   - 数量不符/异常
   - 待办公人员发货
   - 已完成记录
6. 按角色调整每个视图显示字段。
7. 根据需要配置自动化提醒。
8. 邀请办公人员、仓库人员和主管进入表格。
9. 先用 3 至 5 条历史记录测试状态流转是否正确。
10. 测试无误后正式使用。
